# PRD — ClawNet Memory + Telegram Simulation Upgrade

## Overview

Add persistent memory and Telegram simulation features to improve OpenClaw intelligence and remote monitoring.

The goal is:

* reduce hallucinations using memory-backed reasoning
* improve repeated threat detection using historical context
* simulate Telegram status updates for testing and demos

---

# Feature 1 — SuperMemory (Persistent Security Memory)

## Feature

Add memory so OpenClaw remembers what it has previously seen instead of reasoning from scratch every time.

This helps reduce hallucinations and improves repeated threat detection.

If the same suspicious process, IP, file path, ASN, or behavior appears again, ClawNet should recognize it instantly.

---

## Example

Instead of:

```text id="rllb9x"
“node.exe looks suspicious”
```

it should say:

```text id="r4c3cq"
“node.exe connected to this foreign IP 3 times
in the last 7 days and was previously marked suspicious”
```

---

## What to Store

Memory entries for:

* suspicious IPs
* repeated foreign connections
* dangerous ports
* flagged processes
* suspicious file paths
* unsigned binaries
* repeated DNS anomalies
* past kill/block decisions
* previous user approvals/rejections

---

## Implementation

Use:

* PostgreSQL for persistent history
* Redis for fast recent memory cache
* vector memory (optional) for semantic recall

Create:

## Memory Engine

Functions:

* store_event()
* retrieve_similar_events()
* risk_history_lookup()
* prior_decision_lookup()

OpenClaw should query memory before generating analysis.
