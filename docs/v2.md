# PRD — ClawNet v2

## Project Name

**ClawNet — AI-Powered Interactive Security Terminal**

## Overview

ClawNet is an upgraded version of NetWatch powered by OpenClaw.

It provides real-time terminal-based network monitoring, live threat intelligence, AI-powered analysis, safe remediation actions, Telegram threat alerts, and Claude Code plugin support.

The goal is to move from passive monitoring to an interactive AI security system inside the terminal.

---

# Core Features + Implementation

---

## 1. Paginated Live Connection Monitoring

### Feature

Show only 10 active connections at a time instead of printing 200+ rows.

Users should be able to scroll through connections inside terminal.

### Implementation

Use:

* `rich.Live`
* `rich.Table`
* `rich.Layout`
* `rich.Panel`

Add:

* page navigation (`j/k`, arrows, PgUp/PgDn)
* live refresh without flickering
* page indicator

Example:

```text
Showing 1–10 of 241 connections | Page 1/25
```

---

## 2. Real-Time Statistics Panel

### Feature

Statistics should stay visible and update live instead of only showing after `Ctrl+C`.

### Implementation

Continuously update:

* HIGH / MED / LOW risk summary
* Connection status summary
* Top active processes
* Most suspicious processes

Use a fixed stats panel inside terminal layout.

Refresh every few seconds.

---

## 3. Live OpenClaw Intelligence Panel

### Feature

OpenClaw should continuously analyze suspicious activity during runtime.

Not after execution stops.

### Implementation

Run background analysis for:

* suspicious outbound traffic
* foreign IP connections
* unsigned binaries
* abnormal DNS requests
* dangerous ports
* malware-like traffic patterns

Display:

* reason
* severity
* suggested action

inside a live rolling intelligence panel.

---

## 4. Embedded AI Chatbot Interface

### Feature

Add a chatbot section inside terminal below intelligence panel.

User can ask security questions and request actions.

### Example

```text
Why is node.exe suspicious?
Kill PID 4120
Close port 4444
Show all foreign IPs
```

### Implementation

Use:

* OpenClaw runtime
* `prompt_toolkit`
* command parser
* tool execution layer

Terminal layout:

```text
Top → Connection Table
Middle → Stats + Intelligence
Bottom → AI Chat Interface
```

---

## 5. Tool Calling + Safe Actions

### Feature

AI should be able to execute actions safely.

### Allowed Actions

* kill process
* suspend process
* close suspicious ports
* block IPs
* quarantine files
* inspect suspicious files

### Important Rule

If file removal is needed:

## Move to Recycle Bin

Never permanently delete directly.

### Implementation

Use:

* `psutil`
* `send2trash`
* `subprocess`

Require approval for:

* system process termination
* firewall rules
* critical file actions

---

## 6. Telegram Threat Alerts

### Feature

If suspicious or high-risk activity is detected, OpenClaw sends Telegram alerts instantly.

### Implementation

Use:

* Telegram Bot API
* `python-telegram-bot`

`.env`

```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
```

Trigger alerts only for:

* MEDIUM
* HIGH
* CRITICAL

Include:

* process name
* PID
* IP
* country
* reason
* suggested action

---

## 7. Telegram Approval Workflow

### Feature

Allow user to approve or deny actions remotely from Telegram.

### Example

```text
/approve 4120
/deny 4120
/kill 4120
/block 149.154.166.110
/status
```

### Implementation

Create command handlers inside Telegram bot.

Map approvals to action queue inside ClawNet.

High-risk actions wait for approval before execution.

---

## 8. Claude Code Plugin Support

### Feature

Support Claude Code plugin for deeper threat investigation and advanced workflows.

### Implementation

Allow Claude to inspect:

* process trees
* startup persistence
* suspicious files
* shell history
* registry activity
* network anomalies

Claude can help generate:

* detection rules
* remediation steps
* security workflows

Add secure context passing + approval boundaries.
